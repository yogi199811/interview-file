function findDuplicates(arr) {
  const seen = new Set();
  const duplicates = new Set();

  for (const num of arr) {
    if (seen.has(num)) {
      duplicates.add(num);
    } else {
      seen.add(num);
    }
  }

  return Array.from(duplicates);
}

const inputArray = [1, 4, 8, 9, 4, 7, 9];
const result = findDuplicates(inputArray);
console.log(result); // Output: [4, 9]
Explanation:
seen: A Set to store the elements that have been encountered in the array.
duplicates: A Set to store the duplicate elements.
The function iterates through the array, and for each element:

If the element is already in the seen set, it is added to the duplicates set.
If not, it is added to the seen set.
Finally, the duplicates set is converted to an array and returned. Using Sets ensures that each element is stored only once, which efficiently handles the detection of duplicates.

++++++++++++++++++++++++++ Event bubling and and Event Capturing
Ans )  Bubbling
Bubbling is the default way that events propagate. It starts with the element that triggered the event and then bubbles up to the containing elements in the hierarchy.
 This means that event handlers attached to the target element and its ancestors will be called, in the order in which they were attached.

Capturing
Capturing is the opposite of bubbling. It starts with the outermost element and then propagates down to the inner elements.
 This means that event handlers attached to the ancestors of the target element will be called, in the reverse order in which they were attached.

::: what is event propgation ?


1️⃣ Event Propagation
📌 Definition

Event propagation is the mechanism by which an event travels through the DOM tree, starting from the top (capturing), reaching the target element, and then bubbling back up to the top.

🔹 Phases of Event Propagation

Capturing Phase – Event travels from root → target

Target Phase – Event reaches the actual element

Bubbling Phase – Event bubbles from target → root (default)

🔹 Example
child.addEventListener("click", () => {
  console.log("Child clicked");
});

parent.addEventListener("click", () => {
  console.log("Parent clicked");
});

2️⃣ stopPropagation()
📌 Definition

stopPropagation() is a method that stops an event from propagating further through the DOM hierarchy.

🔹 Example
child.addEventListener("click", (e) => {
  e.stopPropagation();
  console.log("Child clicked");
});

++++++++++++++++++++++++++++++++++++++++++Understanding React.memo for Performance Optimization

Ans) React.memo is a higher-order component (HOC) provided by React for optimizing functional components by memoizing them. When a component is wrapped with React.memo, React renders the component and memorizes the result. Before the next render, if the new props are the same, React reuses the memoized result, skipping the next rendering.

Code Example

Here's a basic example of how to use React.memo:

import React, { memo } from 'react';

const MyComponent = ({ name }) => {

console.log('Rendering:', name);

return

 <div>

Hello, {name}!

</div>;

};

export default memo(MyComponent);

In this example, MyComponent will only re-render if the name prop changes. If the parent component re-renders but the name prop remains the same, MyComponent will not re-render, thus saving performance costs.

+++++++++++++++++++++++++++++++++++++++++++++Uncontrolled component: 

An uncontrolled component is a React component where form input values are handled by the DOM itself, not by React state.

In an uncontrolled component, the value of the input element is handled by the DOM itself. Input elements inside uncontrolled components work just like normal HTML input form elements.

The state of the input element is handled by the DOM. Whenever the value of the input element is changed, event-based callbacks are not called. Basically, react does not perform any action when there are changes made to the input element.


import { useRef } from "react";

function UncontrolledForm() {
  const inputRef = useRef();

  const handleSubmit = () => {
    console.log(inputRef.current.value);
  };

  return (
    <>
      <input type="text" ref={inputRef} />
      <button onClick={handleSubmit}>Submit</button>
    </>
  );
}

✅ Advantages

Less code

Simple use cases

❌ Disadvantages

Harder validation

Less control

Not reactive

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++ Error Boundaries

Ans)  A JavaScript error in a part of the UI shouldn’t break the whole app. To solve this problem for React users, React 16 introduces a new concept of an “error boundary”.

Error boundaries are React components that catch JavaScript errors anywhere in their child component tree, log those errors, and display a fallback UI instead of the component tree that crashed. Error boundaries catch errors during rendering, in lifecycle methods, and in constructors of the whole tree below them.

Note

Error boundaries do not catch errors for:

Event handlers (learn more)

Asynchronous code (e.g. setTimeout or requestAnimationFrame callbacks)

Server side rendering

Errors thrown in the error boundary itself (rather than its children)

++++++++++++++++++++++++++++++++++ Hoisting in js

Ans) Hoisting is a JavaScript mechanism where variable and function declarations are moved to the top of their containing scope during the compilation phase. This allows variables and functions to be used before they are declared in the code.



Variable Hoisting

In JavaScript, variable declarations using var are hoisted to the top of their scope and initialized with undefined. This means you can use a variable before it is declared, but its value will be undefined until the assignment is executed.

console.log(foo); // undefined

var foo = 'bar';

console.log(foo); // "bar"

Variables declared with let and const are also hoisted, but they are not initialized. Accessing them before declaration results in a ReferenceError due to the temporal dead zone (TDZ).

console.log(bar); // ReferenceError: Cannot access 'bar' before initialization

++++++++++++++++++++++++++++++++++ Callabck

Ans) JavaScript is designed to handle asynchronous programming which allows us to perform multiple tasks at once without blocking the main execution thread. Callbacks are fundamental in this context, as they enable you to execute specific code after an asynchronous task is completed.



What are Callbacks?

A callback is a function that is passed as an argument to another function and is executed after the completion of that main function. In simple terms, a callback function is called at the end of a task to either deliver results or perform an action. You pass this callback function to the main function, and once the main function completes, it invokes the callback to proceed with the next steps.

Why use Callbacks?

Callbacks are used for managing the outcomes of asynchronous tasks without blocking the program’s execution. Asynchronous tasks, like network requests or database queries, take time to finish. If these tasks were synchronous, the program would halt until they were done, resulting in a sluggish user experience.

With callbacks, though, you can keep the program running while these tasks happen in the background. When the task finishes, the callback function handles the result. This ensures the program stays responsive, enhancing the user experience.

+++++++++++++++++++++++++++++++++++++++++++++++++++++++++=The Virtual DOM Mechanism

Ans) 

React creates a virtual representation of the UI as a virtual DOM tree. When the state of the application changes, instead of updating the real DOM directly, React first updates this virtual DOM. The virtual DOM is a node tree that lists the elements, their attributes, and content as objects and their properties.

React then compares the updated virtual DOM with a snapshot of the virtual DOM taken before the update—a process known as "diffing." During this process, React identifies which virtual DOM objects have changed and updates only those objects on the real DOM. This selective updating prevents unnecessary re-renders and improves performance, especially in complex applications with frequent state changes.









+++++++++++++++++++++++++++++++++DOM Diffing :

Ans) Whenever there is a change in the state of the UI elements, a new virtual DOM is created. Then the new virtual DOM and the previous virtual DOM are compared with each other. This comparing is called DOM diffing.

The intention is to perform minimal operations on the real DOM, hence after diffing, the best way to update the real DOM is calculated, leading to an efficient update of the UI.

++++++++++++++++++++++++++ Event bubling and and Event Capturing

Capturing

Capturing is the opposite of bubbling. It starts with the outermost element and then propagates down to the inner elements.

 This means that event handlers attached to the ancestors of the target element will be called, in the reverse order in which they were attached.

++++++++++++++++++++++++++++++ CALL APPLY AND BIND 

ANS) Absolutely! Understanding call, apply, and bind in JavaScript can be quite empowering. These methods are used to control the context (this value) in which a function is executed. Here's a brief overview of each:



call

The call method calls a function with a given this value and arguments provided individually.

function greet(greeting, punctuation) {

  console.log(greeting + ', ' + this.name + punctuation);

}

const person = { name: 'Alice' };

greet.call(person, 'Hello', '!');

In this example, greet.call(person, 'Hello', '!') sets this to person and passes 'Hello' and '!' as arguments to the greet function.





apply

The apply method is similar to call, but it takes an array of arguments instead of listing them individually.

function greet(greeting, punctuation) {

  console.log(greeting + ', ' + this.name + punctuation);

}

const person = { name: 'Alice' };

greet.apply(person, ['Hello', '!']);

Here, greet.apply(person, ['Hello', '!']) sets this to person and passes the array ['Hello', '!'] as arguments to the greet function.





bind

The bind method creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.

function greet(greeting, punctuation) {

  console.log(greeting + ', ' + this.name + punctuation);

}

const person = { name: 'Alice' };

const greetPerson = greet.bind(person, 'Hello');

greetPerson('!');

In this example, greet.bind(person, 'Hello') creates a new function greetPerson where this is bound to person and the first argument is set to 'Hello'. When greetPerson('!') is called, it logs 'Hello, Alice!'.

1. Scenario: Optimizing a React App

Question: Your React application is slow, and the page re-renders unnecessarily. How would you identify and fix the issue?



Answer:



Identification:

Use React Developer Tools to monitor components that re-render unnecessarily.

Analyze the React Profiler to track rendering performance and identify bottlenecks.

Check dependencies of useEffect, useMemo, or useCallback to avoid redundant executions.

Fixes:

Implement React.memo for functional components to prevent re-renders when props haven’t changed.

Use useCallback to memoize event handlers and useMemo to cache expensive calculations.

Optimize state management by lifting state up or breaking it into smaller, localized states.

Ensure key props are unique in lists to avoid unnecessary DOM operations.

Use virtualization libraries like react-window or react-virtualized to only render visible rows in the viewport.

Paginate the data and fetch it in chunks, loading only the required data for each page.

Memoize components or rows with React.memo to prevent re-rendering unchanged rows.

Debounce or throttle any expensive filtering or search logic.





3. Scenario: Managing Component State

Question: You have a form with multiple fields. Users report that typing into the fields is slow. How would you address this?



Answer:



Avoid storing the entire form state in a single parent component to prevent unnecessary re-renders.

Use useReducer or useState for managing each field's state independently or batch updates using a single handler.

Debounce expensive validation logic in fields using a library like lodash.debounce.

Use controlled components for form fields but ensure the component updates are scoped appropriately.





4. Scenario: Memory Leaks in React

Question: Users report that the application slows down over time. How would you detect and prevent memory leaks in your React app?



Answer:



Detection: Use Chrome DevTools or memory profiling tools to track retained components and event listeners.

Prevention:

Clean up effects in useEffect by returning a cleanup function to remove subscriptions or listeners.

Avoid keeping large objects or closures in state unnecessarily.

Use AbortController for cleaning up ongoing fetch requests.

Ensure components are unmounted properly (e.g., avoid holding onto stale refs or event listeners).





5. Scenario: Error Handling

Question: A component in your app crashes during rendering. How would you handle the error gracefully?



Answer:



Use Error Boundaries to catch errors in the React component tree and display a fallback UI instead of breaking the entire application:

jsx

Copy

Edit

class ErrorBoundary extends React.Component {

 constructor(props) {

  super(props);

  this.state = { hasError: false };

 }



 static getDerivedStateFromError(error) {

  return { hasError: true };

 }



 componentDidCatch(error, errorInfo) {

  console.error("Error caught:", error, errorInfo);

 }



 render() {

  if (this.state.hasError) {

   return <h1>Something went wrong.</h1>;

  }

  return this.props.children;

 }

}

Wrap critical components with the ErrorBoundary.
