SECTION 1: Node.js
------------------

1. What is Node.js?
Definition:
Node.js is a JavaScript runtime built on Chrome's V8 engine that allows running JavaScript on the server side.

Example:
const http = require("http");
http.createServer((req, res) => {
  res.end("Hello World");
}).listen(3000);

2. What is the Event Loop?
Definition:
The event loop handles asynchronous operations in Node.js and allows non-blocking I/O.

Example:
console.log("Start");
setTimeout(() => console.log("Async Task"), 0);
console.log("End");

SECTION 2: Express.js
--------------------

3. What is Express.js?
Definition:
Express is a lightweight Node.js framework used to build APIs and web applications.

Example:
const express = require("express");
const app = express();
app.get("/users", (req, res) => {
  res.json({ name: "Yogesh" });
});

4. Middleware in Express?
Definition:
Middleware functions execute before the request reaches the controller.

Example:
app.use((req, res, next) => {
  console.log("Request received");
  next();
});

SECTION 3: REST API
------------------

5. What is REST?
Definition:
REST is an architectural style for designing networked applications using HTTP methods.

Methods:
GET, POST, PUT, PATCH, DELETE

Example:
GET /api/users → Fetch users
POST /api/users → Create user

SECTION 4: MongoDB & Mongoose
-----------------------------

6. What is MongoDB?
Definition:
MongoDB is a NoSQL, document-based database that stores data in JSON-like format.

Example Document:
{
  name: "John",
  email: "john@gmail.com"
}

7. What is Mongoose?
Definition:
Mongoose is an ODM library for MongoDB in Node.js.

Example:
const userSchema = new mongoose.Schema({
  name: String,
  email: String
});

SECTION 5: Authentication & Authorization

8. What is Authentication?
Definition:
Authentication verifies who the user is.

9. What is Authorization?
Definition:
Authorization determines what the user can access.

10. JWT Authentication?
Definition:
JWT is a token-based authentication mechanism.

Example:
jwt.sign({ id: user._id }, SECRET_KEY, { expiresIn: "1d" });

SECTION 1: Node.js (Basics + Internals)
--------------------------------------
1. What is Node.js?

✅ Definition

Node.js is a JavaScript runtime built on Chrome’s V8 engine that allows us to run JavaScript outside the browser, mainly on the server side, in a non-blocking and event-driven way.

🌍 Real-World Example

In a CRM or inventory system, Node.js is used to:

Handle thousands of API requests

Manage user authentication

Communicate with databases (MongoDB)

Serve REST APIs efficiently

💻 Code Example
const http = require("http");

const server = http.createServer((req, res) => {
  res.end("Hello from Node.js server");
});

server.listen(3000);


2. How Node.js works internally?

✅ Definition

Node.js works using:

Single-threaded event loop

Non-blocking I/O

Async callbacks

Heavy tasks (file, DB, network) are handled by the libuv thread pool, not the main thread.

🌍 Real-World Example

When 100 users hit the login API:

Node.js does not create 100 threads

Requests are queued

DB operations run in background

Responses are returned when ready

💻 Flow (Interview explanation)
Request → Event Loop → Async Task → Callback Queue → Response



3. What is V8 engine?

✅ Definition

V8 is Google’s JavaScript engine that converts JavaScript code into machine code so it can be executed very fast.

🌍 Real-World Example

When you write backend logic (JWT verification, validations),
V8 compiles it and executes it efficiently on the server.

💻 Code Example
// This JS code is compiled by V8 to machine code
const sum = (a, b) => a + b;
console.log(sum(5, 10));


4. What is the Event Loop?
✅ Definition

The Event Loop is the mechanism that allows Node.js to handle multiple asynchronous operations using a single thread.

🌍 Real-World Example

While one API is fetching data from MongoDB,
Node.js can handle other API requests instead of waiting.

💻 Code Example
console.log("Start");

setTimeout(() => {
  console.log("Async Task");
}, 0);

console.log("End");

6. Blocking vs Non-blocking code
Definition

Blocking → Stops execution until task finishes

Non-blocking → Continues execution while task runs in background

🌍 Real-World Example

Blocking DB calls would freeze the server.
Non-blocking allows handling multiple users simultaneously.

💻 Code Example
// Non-blocking
fs.readFile("data.txt", "utf8", (err, data) => {
  console.log(data);
});

console.log("This runs first");
